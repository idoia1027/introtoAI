AI 生成式应用的底层原理

我们用大模型（LLM）比如 GPT、Claude、Gemini，其实是利用了模型的几个核心能力：

理解自然语言的能力（NLU）

能听懂你说的话

生成自然语言的能力（NLG）

能回答、有逻辑、有上下文

结构化执行能力

能调用 API、写代码、操作数据库等

多模态能力（部分模型）

能看图、写图、听声音、生成音频等

AI 应用的本质

是给 LLM 一个"任务包装"与"运行环境"，让它变成更像"工具"的助手，而不是只有对话功能的聊天机器人。

Gemini、DeepSeek、Claude 这些算什么？

它们都是大语言模型（LLM）产品或平台，你可以简单理解成是"AI 大脑"的不同厂家。

名称 背景 特点 是否能当 Agent
Gemini Google 出品 多模态强（图像、视频等），有 Gemini 1.5 Pro 长上下文 可以，但 CLI 是工具
DeepSeek 国内的 AI Lab（深度求索） 支持代码能力突出（DeepSeek-Coder） 本体不是 agent
Claude Anthropic 出品 上下文极长，逻辑稳定性好，文本交互舒适 Claude code 是偏代码助手
GPT OpenAI 出品 生态最丰富，插件最多，GPTs/Agents 支持强 有专门的 GPT Agent 框架

它们本身都是"大脑"，但只是大脑不够用，还得让它做事、接指令、用工具、存储记忆，这就引出了下一个概念。

什么是 Agent？它和直接对话有啥区别？

Agent 的定义：

Agent 就是：一个具备自主性 + 工具能力 + 长期目标执行能力的 AI 程序，围绕某个"角色"或"任务"设计，比如旅游顾问、数据分析员、写作助手。

Agent 是在 LLM 上叠了一层逻辑，让它更像一个"能完成多步骤任务的智能体"，而不是你每次都手动告诉它怎么做。

Agent 的组成通常包括：

  • LLM 大模型（如 GPT、Claude）
  • 提示工程（Prompt）：设计好初始角色和任务思维
  • 工具接入能力：比如能调用网页、API、数据库、代码执行环境
  • 记忆系统：支持长期交互和个性记忆
  • 规划执行器（Planner）：能自动分解复杂任务，自己规划下一步要做啥

🌟 直接用户端内对话

"即时对话 + 临时记忆 + 被动式交互"

🌟 使用 Agent

你可以说一句"帮我做个商业调研"，它自己就去搜集资料、总结、汇报，一路干到底，有记忆，有链条。

Agent如何获得"手和眼睛"？

要让Agent真正落地，需要解决与外部系统的连接问题，目前行业主要有三种方式：

🔌 专用协议

如MCP（Anthropic）
标准化协议，适合企业级开发

🧩 开发框架

如LangChain/LLamaIndex
通过代码灵活连接工具链

🛠️ 原生API

如OpenAI Function Calling
模型原生支持的调用方式

为什么介绍MCP？

作为开源协议，MCP演示了如何安全标准化地连接外部系统，但实际开发中可根据需求选择其他方案

MCP 全称是 Model Context Protocol（模型上下文协议）

MCP 是 Anthropic 开源的一个标准协议，用于连接 AI 助手与各种数据源和工具系统。它解决了一个核心问题：如何让 AI 模型安全、标准化地访问外部资源。

MCP 的核心功能：

  • 统一接口：为不同的数据源和工具提供标准化的连接方式
  • 安全访问：通过协议层面控制 AI 对外部系统的访问权限
  • 上下文管理：高效管理和传递大量外部信息给 AI 模型
  • 工具调用：让 AI 能够调用各种外部 API 和服务

连接数据库

SQL、NoSQL 等数据源

文件系统访问

本地文件、云存储等

API 集成

第三方服务接口

开发工具

Git、IDE、终端等

为什么重要？

MCP 让 AI 助手（如 Claude）能够安全、标准化地连接各种外部系统，这是构建真正实用 AI Agent 的基础设施。它相当于给 AI 提供了"标准化的手和眼睛"来感知和操作外部世界。

Gemini CLI、Claude code 是什么？算 Agent 吗？

不算 Agent，本质是模型的"使用接口"或"编程入口"。

名称 定义 是否 Agent
Gemini CLI Google 出品的命令行工具，用来直接调用 Gemini 模型，适合开发者写代码交互 ❌
Claude Code Claude 在 Code 模式下的代码优化能力 ❌
OpenAI CLI 用命令行调用 GPT 接口的工具 ❌

这些都是"人用 AI"，而不是"AI 自主完成任务"，所以不能算 Agent。

搭 Agent 一般是怎么回事？

如果想搭 Agent，一般包括几个步骤：

步骤 内容 举例
1. 选模型 用什么 LLM，比如 GPT-4、Claude、Gemini 用 OpenAI GPT-4-1106-preview
2. 设计角色/Prompt 定义 agent 是什么角色，有哪些知识、技能 这是一个财务分析师，会提问用户需求，擅长生成图表
3. 工具接入 选择连接方案：
• MCP协议
• LangChain工具包
• 模型原生API
4. 记忆与状态管理 存储与读取用户历史，避免每次重新介绍 使用 RAG + 向量数据库（如 Weaviate）
5. UI 或入口 用网页、命令行、微信、app 接入 接到网页前端或 Discord Bot

Cursor 和 Claude 能搭 Agent 吗？

Claude 上可以搭 Agent 吗？

结论：目前不行。

Claude（包括 Claude 3.5）虽然特别聪明、上下文强、推理好，但不支持以下能力：

  • ❌ 无法调用插件或外部 API
  • ❌ 没有 Memory（长期记忆）
  • ❌ 无"系统角色"设置接口
  • ❌ 无执行环境（不能调用代码或执行命令）

Cursor 能搭 Agent 吗？

结论：可以！但需自己"动手 + 编写结构"。

Cursor 是一款面向程序员的 AI 编程 IDE，集成了 Copilot、GPT-4、Claude 等模型，可以：

  • 自动生成、补全、解释代码
  • 和你多轮协作开发
  • 调用自定义工具、Run Command

它本身不是 Agent，但它可以作为 Agent 的孵化器或执行容器。

创建一个 Python 项目
接入 OpenAI API
使用 LangChain / CrewAI / AutoGen 构建 Agent
在 .env 里配置密钥
逐步扩展自动化任务

什么是 IDE？

IDE（Integrated Development Environment集成开发环境）= 写代码 + 调试 + 项目管理 + 插件 + AI 协作的工作站

一个现代 IDE 通常包括以下几个模块：

模块 作用
代码编辑器 主战场，用来写代码，带有高亮、补全、格式化
调试器（Debugger） 可以逐步执行代码、设置断点、观察变量
终端（Terminal） 执行系统命令或运行程序，比如 python run.py
项目管理器 显示文件结构、依赖、配置文件等
插件系统 可以接入 LLM、Git、Docker、语法检查等工具
版本控制 和 Git 等系统集成，方便代码同步/回滚
构建工具 编译、打包项目，一键运行

常见 IDE： VS Code（最全能，支持插件生态）、Cursor（专为 AI 编程优化）、PyCharm（专业 Python）、Jupyter（数据分析）、IntelliJ IDEA / Xcode（Java/iOS）

🎬 实战展示：YouTube视频分析助手Agent

一个支持输入 YouTube 链接即可获取转录与总结的轻量 Agent，适合内容学习、会议整理、创作参考。

▲ 实际运行界面截图：输入链接 → 显示转录 → 一键总结

✅ 核心原理与结构化流程解析

🎯 设计目标：
搭建一个 输入 YouTube 链接 → 自动转写 → 智能总结 的 Agent，替代手动看视频记笔记的流程，适配学习、会议、创作等场景。

🧠 核心技术组件

步骤功能模块工具/技术
1️⃣下载音频yt-dlp 提取视频音频
2️⃣音频转文字Whisper / faster-whisper 本地模型转写
3️⃣文字理解与总结本地 LLM：如 Deepseek、Mistral、Gemma 等
4️⃣任务流程控制Python 脚本 / LangChain / LangGraph

🔧 模块流程化设计

Agent 通过模块化串联，完成一个完整链路：

📥 Step 1：下载音频

使用 yt-dlp 工具提取视频音频（.mp3 或 .m4a 格式）

🎧 Step 2：音频转文字

本地部署 Whisper 或 faster-whisper 模型，转录为字幕或纯文本

🧠 Step 3：智能总结

调用本地 LLM（如 Deepseek、Mistral、Gemma）提炼要点，生成摘要

🛠️ Step 4：流程调度

使用 Python 脚本或 LangGraph/LangChain 编排任务链，实现自动执行

为什么它是 Agent 而不仅是工具？

  • ✔️ 拥有完整任务链路的自动执行能力
  • ✔️ 能自动决策、串联模块、适配不同输入
  • ✔️ 可复用为其他 Agent 的子流程，例如：会议纪要助手、课程整理工具
  • ✔️ 具备一定"智能协作"能力，而非只能单步被调用
  • ✔️ 在 IDE（如 Cursor）中可持续迭代、可调试部署

给非技术人一个"Agent"的类比式理解

✅ 类比一：从实习生 → 成手助理

聊天机器人像"懂点事但啥都得你教"的实习生；

Agent "你交待目标，它自己能拆事、去找资料、写文档的助理"。

✅ 类比二：从GPS导航 → 高德"打车+订酒店+景点推荐"

简单的 LLM 就是帮你导航某段话或任务；

Agent 你说"我要去厦门玩"，直接帮你把行程安排、酒店预订都搞定。

你是否需要搭 Agent？

你遇到的情况 是否适合搭 Agent 建议
需要自动化、定时任务 ✅ 很适合 让 Agent 替你干活
多步骤长链任务，重复性强 ✅ 很适合 比手动交互省事
临时写代码 / 画图 / 分析数据 ❌ 用对话就够 即时交互更高效
不懂代码又想玩玩 ✅ GPTs 或无代码工具 轻量试水
喜欢精细控制和定制 ✅ 搭建 Agent 框架 LangChain/CrewAI 都可
